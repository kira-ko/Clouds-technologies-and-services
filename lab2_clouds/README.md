# Классификация сервисов Microsoft Azure
После заполнения таблицы классификации стало возможным провести анализ потребления облачных сервисов Microsoft Azure от общего уровня к более детальному, аналогично тому, как это было сделано в лабораторной работе №1 для Amazon Web Services

### 1. Уровень IT Tower (самый общий уровень)
В результате классификации все строки биллинга Azure были распределены по следующим основным IT Tower:
* Compute — вычислительные ресурсы
* Data — работа с данными (хранилища, аналитика, базы данных)
* Networking — сетевые сервисы
* Security — сервисы безопасности
* Cloud Services / Application Services — управляемые облачные сервисы и платформы

### 2. Детализация внутри IT Tower Compute

**Azure Virtual Machines**

Сервис предоставляет виртуальные серверы, что полностью соответствует модели IaaS. Это прямой аналог EC2 / Lightsail из AWS

### 3. Детализация IT Tower Data (работа с данными)

**Хранилища данных**

Azure Data Lake Storage и Azure Blob Storage были отнесены к IT Tower Data, так как используются для хранения и обработки данных. Логика полностью совпадает с классификацией Amazon S3 и Glacier в ЛР1

**Аналитические сервисы**

Сервисы аналитики (например, Azure Analysis Services, Databricks, Data Factory) были классифицированы как аналитические. Это соответствует подходу, который ранее применялся к Amazon Redshift и Amazon Athena

### 4. Сетевые сервисы (IT Tower Networking)

**Azure Traffic Manager**

Сервис используется для маршрутизации пользовательского трафика на уровне DNS. По функционалу и логике потребления он сопоставим с AWS Route 53 и Global Accelerator из первой лабораторной.

### 5. Сервисы безопасности (IT Tower Security)

**Azure Firewall**
Azure Firewall был отнесён к IT Tower Security, так как отвечает за защиту сетевого трафика. Это соответствует подходу классификации сервисов безопасности AWS, таких как AWS WAF и Security Hub


# Вывод
В результате выполнения анализа потребление облачных сервисов Microsoft Azure было успешно распределено по иерархии от уровня IT Tower до Service Usage Type. Это позволило провести анализ потребления от крупных категорий к конкретным типам использования ресурсов, аналогично лабораторной работе №1

Применение единой классификационной модели показало, что несмотря на различия в названиях и реализации сервисов, их функциональное назначение позволяет использовать универсальную кросс-провайдерную модель, пригодную для анализа затрат, архитектуры и управления мультиоблачной (облачной) средой
